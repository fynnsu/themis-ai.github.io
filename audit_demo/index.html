Themis AI Audit Report 

Summary

We ran our end-to-end debiasing platform on an imbalanced MNIST dataset. This tutorial goes over the report output of our platform.

This imbalanced dataset consists of only ~60 samples of even digits and ~3000 samples of odd digits. The challenge here is to train a model, given this imbalanced dataset, such that it learns even digits as well as odd digits. The report is divided into two segments. The first segment is the output of our Bias Audit, and the second segment is the output of our Debiased Training. Throughout this audit report, you will gain a better understanding of the data distribution, can easily catch noisy data samples, and compare the performance of a baseline model with a debiased model.

Bias Audit

The very first step of any ML design and implementation is to understand the dataset you are working with. With our Bias Audit, you can dig deeper into your dataset, understand data distribution, identify different underlying features within the dataset and features within each class, in addition to providing the distribution of such underlying features.

This report consists of three sections, Class Distribution, Feature Clusters, and Features within Class.

Class Distribution

Class Distribution plots demonstrate how different classes within your dataset are distributed. This is the very first step of identifying under-represented classes within your dataset.

Feature Clusters

Our state-of-the-art algorithm identifies underlying features within the dataset and clusters the data based on them. This tool can be utilized as a dataset debugger (to identify noisy data samples) and to identify underrepresented features. The same plot is presented in three different forms, colored by cluster, colored by label, and each class in isolation, to further assist you in studying the data.

Features within Class

In this section, our algorithm has identified underlying features within each class. The plots demonstrate the relative distance of such features and their difficulty level for a prediction model. These plots can be utilized as another dataset debugging tool that enables a deeper understanding of data samples within each class.
